# Markdown正規化プロジェクト

目的: 全記事（1874件）を Hugo に適したシンプルな Markdown に統一する。
過去のプラットフォーム移行（旧ブログ → Medium export → Medium RSS自動インポート）の変換痕跡を除去する。

> **このドキュメントは2026-07-10に完了した一括正規化（Phase 1〜8）の記録。**
> 未完了の課題と新しく見つかる課題は [GitHub Issues](https://github.com/nutsfall/blog.qli.jp/issues) で管理する。

## 運用ルール（完了時点のもの）

- 修正はフェーズ単位でサブエージェント（Sonnet）が実施する
- 件数が多い課題は、ファイルを直接編集せず **スクリプトを書いて一括適用 → サンプル検証** する
- 修正前後で `scripts/content_lint.zsh` を実行し、対象課題の件数が減り、他の課題の件数が変わっていないことを確認する
- フェーズ完了ごとに `hugo build` で確認し、レビュー後にコミットする
- コンテンツ作業中に新しい課題を見つけた場合は GitHub Issue にする
