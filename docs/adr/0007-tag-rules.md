# ADR-0007: タグ規約（英語小文字・1〜3個・具体的）とauto_taggerへの一元化

- ステータス: 採用
- 日付: 2026-05-15（全1864記事へ一括適用）（2026-07-13 事後記録）

## コンテキスト

旧記事はタグがない、またはMedium由来の不揃いなタグを持っていた。タグページを機能させるには語彙の統制が必要。

## 決定

タグは以下の規約に統一し、全記事へ一括付与した（コミット c90fdba4）:

- 英語・小文字、複数語はハイフン区切り（`j-league`, `apple-music`）
- 1記事1〜3個（短い記事は1個で可）
- 汎用語（`diary`, `misc`, `thoughts`, `life`）より具体的な語を優先
- 製品・アーティスト・チーム・作品は固有名詞を使う（`apple`, `netflix`, `avispa-fukuoka`）

タグの追加・修正は手動でfrontmatterを編集せず、必ず `scripts/auto_tagger.rb` を通す。Mediumインポートのタグは規約外のため常に再付与する。分布確認は `scripts/tag_checker.rb`。

## 結果・影響

- タグ語彙が安定し、既存タグとの一貫性チェックが機能する
- タグ付けはLLM（Claude）依存。その実行方式は後に変更した（→ ADR-0010）
