# ADR-0006: caption自動検出はヒューリスティック維持、誤検出は手動修正

- ステータス: 採用
- 日付: 2026-07-10（2026-07-13 事後記録）

## コンテキスト

Mediumのmarkdown変換後は、画像キャプションと本文段落が同じプレーンテキスト構造になり、確実な判別が原理的に不可能。`process_new_posts.rb` は「画像直後の200文字以内の行」をcaptionと判定するが、本文冒頭の段落を誤ってcaptionにすることがある（実例: 2026-07-13「モノクロだけのカメラ」）。

## 決定

- 新規インポート（`process_new_posts.rb`）の自動検出ロジックは現状のまま維持し、**誤検出は手動でcaptionを削除して本文に戻す**。captionが必要な場合も手動で追記する
- 旧記事の一括処理（`cover_migrator.rb`）に限り「全角句読点（。、！？）を含む行はcaptionにしない」叙述文除外を追加した（Phase 7の実データ51件で検証: 真正キャプション2件=Flickrクレジットは通過、誤検出49件は全て排除）

## 結果・影響

- 新規インポート後はcaptionの目視確認が必須の運用になる
- 旧記事一括分では誤検出率ほぼ100%だったため叙述文除外が有効だったが、新規記事に同ルールを持ち込むかは未判断（英語captionには効かない）

## 却下した代替案

- 文字数制限を絞る — 誤爆は完全になくならない
- caption検出の廃止 — captionが必要なケースが実在する（Flickrクレジット等）
- イタリック判定 — Mediumのcaptionはプレーンテキストで出力される
- 英語のみ判定 — 日本語captionもある
