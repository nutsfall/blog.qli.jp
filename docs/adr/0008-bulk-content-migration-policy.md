# ADR-0008: 大量コンテンツ修正はスクリプト一括適用＋lint前後比較で行う

- ステータス: 採用
- 日付: 2026-07-10（Markdown正規化プロジェクト Phase 1〜8 完了時に確立）（2026-07-13 事後記録）

## コンテキスト

全1874記事に旧ブログ→Medium→RSSインポートの変換痕跡（崩れたリンク、計測ピクセル、定型ブロック、エスケープ残骸など）が残っていた。手作業では規模的に不可能で、LLMに直接編集させると再現性・検証可能性がない。

## 決定

件数の多いコンテンツ修正は以下の方式で行う:

- ファイルを直接編集せず、**`--dry-run` 付きのスクリプトを書いて一括適用**し、スクリプトは `scripts/` に残す
- 修正前後で `scripts/content_lint.zsh` を実行し、対象課題の件数が減り**他の課題の件数が変わっていない**ことを確認する
- フェーズごとに `hugo build` とdiff目視レビューを通してからコミットする（pushは明示指示があるときのみ）
- 機械判定が危ういもの（レンダリングに影響するエスケープ等）はHTML差分検証で個別判断する
- 作業中に見つけた新しい課題はその場で直さずGitHub Issueにする

## 結果・影響

- Phase 1〜8（frontmatter正規化、リンク・画像・定型ブロック・見出し・description等）を完走。記録は `MIGRATION.md`
- 意図的に**対応しない**と決めたもの: 三点リーダ代用 `･･･`（当時の文体として保存）、レンダリングに影響するエスケープ残骸24件、draft記事2件、ASINなしAmazonリンク1件、独自要約として残したdescription 2件
